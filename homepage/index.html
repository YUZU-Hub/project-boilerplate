Quick Start

Get running in under a minute

1

Clone the repository

git clone https://github.com/your-org/project-boilerplate.git my-project
2

Start development server

cd my-project && docker compose up --build
3

Open in browser

localhost:3000 - Homepage localhost:3001 - Webapp localhost:3002 - Admin localhost:8090/_/ - PocketBase Admin

Project Structure

Clean organization for any size project

project/
├── homepage/          # Landing pages (port 3000)
│   ├── css/
│   ├── js/
│   └── index.html
├── webapp/            # Main application (port 3001)
│   ├── css/
│   ├── js/
│   └── index.html
├── admin/             # Admin dashboard (port 3002)
│   ├── css/
│   ├── js/
│   └── index.html
├── server/            # Node.js Express servers
│   ├── homepage.js
│   ├── webapp.js
│   ├── admin.js
│   └── index.js
├── api/               # PocketBase (port 8090)
│   ├── pb_hooks/      # Custom hooks
│   ├── pb_migrations/ # Schema migrations
│   └── pb_data/       # Database (gitignored)
├── Dockerfile         # Production build
├── Dockerfile.dev     # Development build
├── docker-compose.yml # Local development
└── entrypoint.sh      # Process manager

Customize Each Frontend

Each directory (homepage, webapp, admin) is independent. Replace with React, Vue, Svelte, or keep it vanilla.

PocketBase Hooks

Add custom API routes, cron jobs, and event handlers in api/pb_hooks/. Changes hot-reload in development.

Environment Modes

Development uses volume mounts for hot-reload. Production builds everything into a single optimized image.

Deploy to Production

Push to main and let Coolify handle the rest

Coolify (Recommended)

Self-hosted PaaS with automatic SSL, rolling deploys, and built-in monitoring.

  1. Connect your GitHub repo to Coolify
  2. Point to the root Dockerfile
  3. Configure domains for each port
  4. Push to deploy

Any Docker Host

Works on any server that runs Docker.

docker build -t myapp . && docker run -p 3000:3000 -p 3001:3001 -p 3002:3002 -p 8090:8090 myapp
